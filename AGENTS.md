# AGENTS.md — Fortran Codes Documentation

This directory contains Python scripts (see `README.md`) and Fortran programs
for celestial mechanics: orbit determination from backyard observations,
N-body simulation, Kalman filtering, SPK ephemeris I/O, and Artemis II
trajectory processing.

## Key data files

| File | Purpose |
|------|---------|
| `de440s.bsp` | JPL DE440s planetary ephemeris (required by most Fortran programs) |
| `nutation.dat` | Flat-binary nutation coefficients (generated by `convert_nutation_data.py`) |
| `observations.dat` | Synthetic Sun/Moon alt/az observations (generated by `generate_observations`) |
| `artemis_ephemeris.json` | Orion spacecraft positions from JPL Horizons |
| `artemis_orion.dat` | Text conversion of above for Fortran |
| `artemis_trajectory.dat` | Fortran-computed trajectory output |

## Building with fpm

```bash
cd examples/
fpm build --compiler flang        # build all 9 programs
fpm run --compiler flang <name>   # run a specific program
```

Available program names: `compute_from_scratch`, `generate_observations`,
`sun_ekf_kepler`, `sun_ekf`, `moon_ekf`, `sun_ekf_3body`,
`kalman_sim`, `newton_sim`, `artemis_trajectory`.

## Project layout

```
src/   — shared library modules (constants, SPK reader, linalg, nutation,
          astro, obs_reader, nbody2)
app/   — one .f90 per executable
```

## Manual compilation (without fpm)

- `lfortran` — primary compiler for `artemis_trajectory.f90` and `compute_from_scratch.f90`
- `flang` — used for all other programs

## Programs

### `compute_from_scratch.f90`

Pure Fortran translation of `compute_from_scratch.py`.  
Computes Sun and Moon apparent positions (altitude, azimuth, angular diameter,
distance) entirely from scratch, implementing:

- UTC → TT → TDB time scale conversions
- IAU 2000A precession and nutation
- ICRS-to-J2000 bias, Earth rotation angle, sidereal time (GMST/GAST)
- WGS84 → ITRS → GCRS observer position
- Light-travel-time correction (iterative)
- Gravitational light deflection (Sun, Jupiter, Saturn, Earth)
- Stellar aberration (relativistic)
- Horizon (alt/az) coordinate conversion

Results must agree with the Python version to machine precision (~10⁻¹¹°).

**Build and run:**
```bash
fpm run --compiler flang compute_from_scratch
```

**Inputs:** `de440s.bsp`, `nutation.dat`  
**Output:** Prints Sun and Moon positions for the April 8, 2024 total solar eclipse
(Fredericksburg, TX) with validation.

---

### `generate_observations.f90`

Generates synthetic backyard Sun & Moon observations (alt/az) with realistic noise,
for use as input to the Kalman filter programs.

For each day in the configured time span:
- Finds sunrise/sunset and moonrise/moonset by bisection
- Observes at rise, set, and every ~2 hours while above the horizon
- Adds systematic bias + random Gaussian noise (< 1 arcmin mean)

**Build and run:**
```bash
fpm run --compiler flang generate_observations
```

**Inputs:** `de440s.bsp`, `nutation.dat` (observer lat/lon hardcoded in source)  
**Output:** `observations.dat`

---

### `sun_ekf_kepler.f90`

2-body Sun–Earth EKF (Extended Kalman Filter) using Keplerian elements.
Earlier/simpler version of `sun_ekf.f90` — semi-major axis `a` is fixed
(taken from DE440s) rather than estimated.

**State vector (6):** `[e, i, Ω, ω, M₀, μ]`  
All elements are constants of motion in 2-body; dynamics matrix F = I.

**Filter:** EKF with finite-difference Jacobian  
**Propagation:** Yoshida 4th-order N-body integrator

**Build and run:**
```bash
fpm run --compiler flang sun_ekf_kepler
```

**Inputs:** `observations.dat`, `de440s.bsp`  
**Output:** Prints estimated orbital elements and derived physical constants.

---

### `sun_ekf.f90`

2-body Sun–Earth EKF that also estimates the semi-major axis `a`.

**State vector (7):** `[e, i, Ω, ω, M₀, μ, a]`

Like `sun_ekf_kepler.f90` but adds `a` to the state, allowing the filter to
derive the Earth–Sun distance from Sun alt/az observations alone.
Note: for the distant Sun, diurnal parallax is negligible, so the μ–a
degeneracy may not be fully broken by angular observations alone.

**Build and run:**
```bash
fpm run --compiler flang sun_ekf
```

**Inputs:** `observations.dat`, `de440s.bsp`  
**Output:** Prints estimated orbital elements and derived physical constants.

---

### `moon_ekf.f90`

2-body Earth–Moon EKF with N-body propagation.

**State vector (7):** `[e, i, Ω, ω, M₀, μ, a]`

Key physics: the Moon is close enough (~386,000 km) that diurnal parallax
from the observer's surface position (~6378 km) is ~57 arcmin — this directly
constrains the distance `a` and breaks the μ–a degeneracy that exists for the
distant Sun.

**Build and run:**
```bash
fpm run --compiler flang moon_ekf
```

**Inputs:** `observations.dat`, `de440s.bsp`  
**Output:** Prints estimated Moon orbital elements and derived GM values.

---

### `sun_ekf_3body.f90`

3-body Sun–Earth–Moon staged EKF. Most complete orbital determination program.

**State vector (14):**
- `x(1:7)` = `[e_E, i_E, Ω_E, ω_E, M0_E, μ_SE, a_E]` — Earth orbit around Sun
- `x(8:14)` = `[e_M, i_M, Ω_M, ω_M, M0_M, μ_EM, a_M]` — Moon orbit around Earth

**Mass derivation:**
- `GM_earth = μ_EM − GM_moon` (GM_moon fixed at 4902.800066 km³/s²)
- `GM_sun = μ_SE − GM_earth`

**Filter:** Staged EKF with finite-difference Jacobian:
1. Stage 1: Earth orbit from Sun observations
2. Stage 2: Moon orbit from Moon observations
3. Stage 3: Joint refinement from all observations

**Propagation:** Yoshida 4th-order N-body integrator (N=3)

**Build and run:**
```bash
fpm run --compiler flang sun_ekf_3body
```

**Inputs:** `observations.dat`, `de440s.bsp`  
**Output:** Prints estimated orbital elements for Earth and Moon, and derived
GM values for Sun, Earth, Moon.

### `kalman_sim.f90`

Full 3-body (Sun, Earth, Moon) Unscented Kalman Filter (UKF) estimation
from backyard observations.

**State vector (21):**
- `x(1:6)` — Sun position + velocity
- `x(7:12)` — Earth position + velocity
- `x(13:18)` — Moon position + velocity
- `x(19)` — GM_sun
- `x(20)` — GM_earth
- `x(21)` — GM_moon

Works in Cartesian coordinates using the scaled UKF sigma-point method.
Note: sigma points are propagated only for the measurement update; the
prediction step propagates only the mean (N-body orbits diverge too fast
for sigma-point spreading over long intervals).

**Build and run:**
```bash
fpm run --compiler flang kalman_sim
```

**Inputs:** `observations.dat` (noisy alt/az), `de440s.bsp`  
**Output:** Prints estimated GM values and positions over time.

---

### `newton_sim.f90`

Builds a custom Newtonian SPK kernel (`newton.bsp`) by:
1. Reading DE440s initial states at J2010.0
2. Integrating a 10-body Newtonian model with the Yoshida 4th-order scheme
3. Fitting type-2 Chebyshev segments
4. Writing a minimal DAF/SPK binary file

**Build and run:**
```bash
fpm run --compiler flang newton_sim
```

**Inputs:** `de440s.bsp`  
**Output:** `newton.bsp` (custom SPK ephemeris kernel)

---

### `artemis_trajectory.f90`

Computes Artemis II mission trajectory data for plotting.

Reads Orion spacecraft positions (downloaded from JPL Horizons, converted
to text by `convert_artemis_data.py`) and computes Moon position from
`de440s.bsp`. Also reports the maximum Earth distance with MET and
Mountain Daylight Time.

**Build and run:**
```bash
fpm run --compiler flang artemis_trajectory
```

**Inputs:** `artemis_orion.dat`, `de440s.bsp`  
**Output:** `artemis_trajectory.dat` (columns: timestamp, MET_s, Orion xyz km,
Moon xyz km, Earth dist km, Moon dist km, velocity km/s, range rate km/s)

## Data Flow

```
JPL Horizons (web)
    └─ plot_trajectory.py ──→ artemis_ephemeris.json
                                  └─ convert_artemis_data.py ──→ artemis_orion.dat
                                                                       └─ artemis_trajectory ──→ artemis_trajectory.dat
                                                                                                      └─ plot_artemis_fortran.py ──→ artemis_trajectory_fortran.png

de440s.bsp + nutation.dat
    ├─ generate_observations ──→ observations.dat
    │       └─ sun_ekf / moon_ekf / sun_ekf_3body / kalman_sim  (orbit determination)
    ├─ compute_from_scratch  (apparent position validation)
    └─ newton_sim ──→ newton.bsp  (custom SPK kernel)
```
